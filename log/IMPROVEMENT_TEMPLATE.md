---

## 🔄 개선 사이클 #N: [제목]

**테스트 결과:**
- 정확도: XX% (목표 80% 미달)
- 평균 응답 시간: X.Xs
- 주요 실패 항목:
  - [ ] 질문1: 문제 설명
  - [ ] 질문2: 문제 설명

**원인 분석:**
- [ ] 데이터 누락
- [ ] 프롬프트 불명확
- [ ] 검색 알고리즘 문제
- [ ] 응답 시간 초과
- [ ] 기타:

### 2️⃣ 개선 작업 ([시작~종료 시간])

**변경 사항:**

#### A. 프롬프트 수정
```diff
- Before: "기존 프롬프트 내용"
+ After: "새 프롬프트 내용"
```
- 파일: `rag/web.py` (SYSTEM_PROMPT)
- 목적: [왜 수정했는지]

#### B. 데이터 추가/수정
- [ ] 크롤링: [게임/아이템/캐릭터 이름]
  - 파일: `crawler/data/[game]/[파일명].txt`
  - 크기: XX자
  - 출처: [나무위키/palworld.gg 등]

#### C. 알고리즘 튜닝
- [ ] 벡터 가중치: 0.X → 0.Y
- [ ] BM25 가중치: 0.X → 0.Y
- [ ] Top-K: X → Y
- [ ] 청크 크기: X → Y

#### D. 기타
- 설명...

**Git 커밋:**
```bash
git add [파일들]
git commit -m "[커밋 메시지]"
git push origin main
# 커밋 해시: abc1234
```

### 3️⃣ 재테스트 ([시간])

**Before → After 비교:**

| 지표 | Before | After | 변화 |
|------|--------|-------|------|
| 정확도 | XX% | YY% | ✅ +ZZ%p |
| 응답 시간 | X.Xs | Y.Ys | ✅ -Z.Zs |
| 실패 항목 | N개 | M개 | ✅ -K개 |

**개별 항목 개선:**
- ✅ 질문1: [개선 내용]
- ✅ 질문2: [개선 내용]
- ⚠️ 질문3: [아직 미해결]

## 💡 사용 예시

```markdown
## 🔄 개선 사이클 #1: 정확도 향상 (프롬프트 자연스럽게)

### 1️⃣ 문제 발견 (17:28 QA 테스트)
- 정확도: 45%
- 주요 실패: 한조 스킬 누락 (음파 화살), 네더라이트 정보 부족

### 2️⃣ 개선 작업 (17:35~17:50)
**프롬프트 수정:**
- Before: "참고 자료의 정보를 EXACTLY 그대로 전달"
- After: "참고 자료에서 정확히 인용하여 자연스럽게 답변"

**데이터 추가:**
- 네더라이트 페이지 크롤링 (4,823자)

커밋: `8da4ab7`

### 3️⃣ 재테스트 (18:10)
- 정확도: 45% → 62% (✅ +17%p)
- 응답 시간: 26.2s → 18.5s (✅ -7.7s)

### 4️⃣ 결론
✅ 프롬프트 자연스럽게 수정 효과 있음
⚠️ 목표 80% 도달 위해 추가 데이터 품질 개선 필요
```
